# Аналитика закупок группы Сбербанк

## Описание проекта

Проект представляет собой комплексное enterprise-решение для автоматизированного сбора, очистки, нормализации и анализа открытых данных о закупках ПАО Сбербанк и 31 дочернего юридического лица (Sber Group) за 2024–2025 годы.

Система решает следующие задачи:
1. **Сбор данных (Парсинг):** Агрегация тендерных процедур с более чем 10 электронных торговых площадок (Сбербанк-АСТ, B2B-Center, ЕИС и др.) с обходом защиты (Playwright + CAPTCHA bypass).
2. **Entity Resolution:** Строгий Jaro-Winkler match для точной идентификации 32 юрлиц (исключая случаи, когда Сбербанк выступает лишь оператором площадки).
3. **Очистка и Дедупликация:** Кросс-источниковая дедупликация пересекающихся закупок между площадками.
4. **Аналитика и ML:** Применение методов машинного обучения (Isolation Forest, MinHashLSH) для поиска аномалий (дробление лотов, 0% экономии при 1 участнике) и расчёт индекса Герфиндаля-Хиршмана (HHI) для выявления монополизации.

**Итоговый чистый датасет:** 3 161 уникальная закупка на общую сумму 30,5 млрд ₽.

## 1. Запуск презентации и дашборда

Результаты анализа упакованы в интерактивный дашборд на React 19 (Vite + Recharts) и автоматическую защитную речь для презентации.

### Интерактивный дашборд

Дашборд состоит из 5 вкладок (История, Топ закупок, Макроэкономика, ML Аномалии, AI Инсайты) и поддерживает смену светлой/тёмной темы.

```powershell
cd presentation
npm install
npm run dev
```
*После запуска откройте в браузере: http://localhost:5173*

### Генерация защитной речи

Для защиты тестового задания генерируется готовый документ `Defense_Speech.docx` с текстом для выступления по каждой вкладке дашборда.

```powershell
cd presentation
python generate_defense_speech.py
```
*(В репозитории уже лежит `Defense_Speech_FINAL.docx` на случай, если локальная генерация недоступна).*

## 2. Общий пайплайн (Парсинг и Merge)

Вы можете запустить сбор данных со всех площадок как по отдельности, так и автоматизированно. 

### Вариант А. Запуск парсинга по одному источнику

Используется для точечного обновления данных конкретной площадки.

```powershell
# Пример: Сбор данных со Сбербанк-АСТ (production)
python scripts\sberbank_ast_prompt2_source_sprint_v2.py --batch-name ast-manual --inns 7707083893

# Пример: Сбор данных с B2B-Center (production, использует Playwright)
python scripts\b2b_center_prompt2_source_sprint_v2.py --batch-name b2b-manual --inn 7736663049

# Пример: Сбор с ЕИС (control coverage)
python scripts\eis_prompt2_source_sprint_v2.py --batch-name eis-manual --inns 7707083893
```

### Вариант Б. Одновременный запуск основных источников

Чтобы собрать все данные разом, выполните последовательно (или напишите небольшой `.ps1` скрипт):

```powershell
python scripts\sberbank_ast_prompt2_source_sprint_v2.py --batch-name ast-full-2024-2025 --inns all
python scripts\b2b_center_prompt2_source_sprint_v2.py --batch-name b2b-full-2024-2025 --inns all
python scripts\eis_prompt2_source_sprint_v2.py --batch-name eis-full-2024-2025 --inns all
```

### Слияние и дедупликация (Merge)

После парсинга необходимо объединить результаты и удалить дубликаты. Конфигурация того, какие пакеты (`batch`) брать в работу, указывается в `configs/source_sprints_allowlist.csv`.

```powershell
# Проверка перед слиянием (Dry-run) - покажет, сколько дублей будет удалено
python scripts\merge_sprints.py --dry-run

# Боевое слияние (создаст output/merged_sprints.csv)
python scripts\merge_sprints.py
```

## 3. База данных (SQLite)

Для удобной демонстрации SQL-запросов комиссии используется SQLite (не требует установки сервера PostgreSQL). База собирается автоматически из итогового чистого датасета.

```powershell
# Из корня проекта запускаем экспорт:
python export_to_sqlite.py
```
**Результат:** В корне появится файл `purchase_analysis.db`. База содержит таблицу `lots`, таблицу `entity_scope` и 9 подготовленных аналитических VIEW (например, `v_summary`, `v_hhi`, `v_top20`).

**Для демонстрации:**
1. Скачайте и откройте программу **DB Browser for SQLite**.
2. Нажмите `File -> Open Database` и выберите сгенерированный файл `purchase_analysis.db`.
3. Перейдите на вкладку `Execute SQL` и скопируйте запросы из файла `demo_queries.sql` (содержит 15 заготовленных выборок).
4. Нажимайте кнопку Play (`Run`) для демонстрации результатов.
